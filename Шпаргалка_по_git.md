# Шпаргалка по основным командам `Git` а так же Создание файла `.gitignore` в Python-проекте

В этом блокноте собраны наиболее часто используемые команды `Git` для работы с репозиториями. Для удобства каждая команда сопровождается кратким описанием её назначения.

## Основные команды

### Начальная настройка

#### `git config`
Настройка глобальных параметров конфигурации Git. Например, установка имени и email автора коммитов:

```bash
git config --global user.name "Ваше Имя"
git config --global user.email "ваш_email@example.com"
```
### Работа с репозиторием

#### `git init`
Инициализация нового локального репозитория в текущей директории:

```bash
git init
```

#### `git clone`
Клонирование удалённого репозитория на локальный компьютер:

```bash
git clone https://github.com/username/repository.git
```



## Чтобы начать работать с Git и сделать первый коммит, вам нужно выполнить следующие шаги:

### 1. Создайте новый файл или скопируйте существующие файлы в вашу рабочую директорию.
### 2. Добавьте файлы в индекс Git с помощью команды :
```bash
git add.
```
Вы можете добавить конкретный файл, используя команду:
```bash
 git add <имя_файла>
 ```
 Или добавить все файлы в директории с помощью команды:

```bash
git add .
```
### 3. Сделайте коммит изменений с описательным сообщением с помощью команды:
```bash
 git commit -m "Ваше сообщение коммита"
```   
#### `git status`
Проверка статуса рабочего дерева и индекса:

```bash
git status
```

#### `git diff`
Просмотр различий между текущим состоянием и последним коммитом:

```bash
git diff
```

### История и навигация

#### `git log`
Просмотр истории коммитов:

```bash
git log
```

#### `git checkout`
Переключение между ветками или восстановление файлов:

```bash
git checkout branch_name
git checkout -- filename.txt
```

#### `git reset`
Отмена последних изменений:

```bash
git reset HEAD~1
```

### Ветвление и слияние

#### `git branch`
Создание новой ветки:

```bash
git branch new_branch
```

#### `git merge`
Слияние одной ветки с другой:

```bash
git merge other_branch
```

#### `git rebase`
Перемещение коммитов одной ветки поверх другой:

```bash
git rebase master
```

### Удаление локальной ветки
Для удаления локальной ветки выполните следующую команду:

```bash
git branch -d <имя_ветки>
```
Эта команда удаляет указанную ветку, но перед этим проверит, чтобы все коммиты этой ветки были слиты (merged) с другой веткой. Если есть неслитые коммиты, то Git выдаст предупреждение и предложит использовать ключ -D.

Чтобы принудительно удалить ветку без проверки слияния, используйте команду:

```bash
git branch -D <имя_ветки>
```

### Удаление удаленной ветки

Удалённая ветка — это копия ветки на сервере (например, на GitHub, Bitbucket и др.). Чтобы удалить её, нужно выполнить команду:

```bash
git push origin --delete <имя_ветки>
```

Здесь origin — это имя вашего удаленного репозитория (обычно так называется основной удалённый репозиторий). Вы можете заменить origin на другое название, если у вас несколько удалённых репозиториев.


После выполнения этой команды ветка будет удалена с сервера, однако она останется в вашем локальном кэше до тех пор, пока вы не обновите информацию о состоянии удалённых веток командой:

```bash
git fetch --prune
```
Это позволит вам синхронизироваться с текущими изменениями на сервере и убрать ссылки на удалённые ветки, которых больше нет.

###  Извлечение всех файлов из ветки main в текущий рабочий каталог:

```bash
git checkout main -- .
```

Здесь происходит извлечение всех файлов из ветки main в ваш текущий рабочий каталог. Символ точки (.) указывает на текущий каталог, таким образом, команда извлекает файлы из ветки main и помещает их в текущую директорию, которая находится например в ветке master.

Важно отметить, что эта операция не изменяет историю коммитов ветки master. Она просто копирует содержимое файлов из одной ветки в другую.

### Фиксация изменений в ветке master

ветка master это как пример, у вас может быть любое другое название вашей ветки:

```bash
git commit -m "Merged all files from main into master"
```
Теперь, когда все файлы из ветки main находятся в рабочем каталоге ветки master, необходимо зафиксировать эти изменения. Команда commit создает новый коммит в ветке master, добавляя все изменения, сделанные после последнего коммита. Параметр -m позволяет указать сообщение для этого коммита.



### Удалённые репозитории

#### `git remote`
Управление удалёнными репозиториями:

```bash
git remote add origin https://github.com/username/repository.git
```

#### `git remote -v`
Отображения списка удалённых репозиториев, настроенных для текущего локального репозитория Git:

```bash
git remote -v
```

#### `git push`
Отправка изменений на удалённый репозиторий:

```bash
git push origin main
```

#### `git pull`
Получение обновлений из удалённого репозитория:

```bash
git pull origin main
```




# Удаление Файлов из Git Репозитория

Чтобы удалить файлы из вашего локального репозитория Git и навсегда удалить их из истории репозитория, выполните следующие шаги:

## Шаг 1: Удаление файлов из рабочей директории

Удалите файлы, используя команду `git rm`. Замените `<filename>` на имя файла, который вы хотите удалить. Если нужно удалить несколько файлов, выполните эту команду для каждого из них:

```bash
git rm --cached <filename>
```

## Шаг 2: Добавление изменений в индекс

Добавьте изменения в индекс Git с помощью команды:

```bash
git add .
```

## Шаг 3: Создание коммита

Создайте новый коммит, который будет содержать удаление файлов:

```bash
git commit -m "Remove files from repository"
```

## Шаг 4: Окончательное удаление из истории репозитория

Чтобы удалить файлы навсегда из истории репозитория, выполните команду `git push` с опцией `--force`. Замените `<branch name>` на имя ветки, в которой вы работаете:

```bash
git push --force origin <branch name>
```

**Важно**: `git push --force` может привести к потере истории для других разработчиков, которые уже склонировали ваш репозиторий. Убедитесь, что все участники проекта согласны с удалением файлов и обновлением истории, прежде чем выполнять эту команду.

# Инструкция для команды `git mv`

Команда `git mv` используется в Git для переименования файлов внутри репозитория. Она работает аналогично команде `mv` в Unix-подобных системах, но также учитывает историю изменений в Git.

1. **Синтаксис:**

```bash
git mv old_filename new_filename
```

2.**Пример использования:**
   Предположим, у вас есть файл `old_filename` и вы хотите переименовать его в `new_filename`. Выполните следующую команду:

```bash
git mv old_filename new_filename
```


3.**Ожидаемый результат:**
   После выполнения команды `git mv`, файл `old_filename` будет переименован в `new_filename` внутри вашего репозитория `Git`. Это изменение будет сохранено в истории коммитов, и вы сможете отслеживать его как отдельный шаг в процессе разработки.

4.**Дополнительная информация:**
   - Эта команда не изменяет содержимое файла, она просто меняет его имя.
   - Если вы хотите изменить содержимое файла и сохранить это изменение в истории коммитов, сначала выполните `git mv`, а затем внесите изменения в файл и сделайте новый коммит.
   - Команда `git mv` также может использоваться для перемещения файлов между директориями внутри репозитория.


   
## Создание файла `.gitignore` в Python-проекте

Создание файла `.gitignore` в Python-проекте можно выполнить прямо из терминала (командной строки). Вот команда, которая создаст файл `.gitignore` и добавит в него строку для исключения папок виртуального окружения:

```bash
echo .venv > .gitignore
```

Если вы хотите сгенерировать стандартный набор исключений для `Python-проекта`, вы можете использовать следующую команду для получения предустановленного содержимого из сервиса [gitignore.io](https://www.toptal.com/developers/gitignore):

```bash
curl https://www.toptal.com/developers/gitignore/api/python > .gitignore
```

Убедитесь, что у вас установлена утилита `curl` перед использованием этой команды.

Для ручного добавления содержимого в `.gitignore`, откройте файл в текстовом редакторе и введите необходимые строки.
